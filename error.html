使用测试样例查找问题原因

通过test.py进行测试：

import os
import torch
import torch.distributed as dist
from torch.multiprocessing import Process

def run(rank, size):
    tensor = torch.tensor([i for i in range(3)])
    tensor2 = torch.tensor([5])
    if rank == 0:
        tensor += 1
        tensor2 += 2
        # Send the tensor to process 1
        dist.send(tensor=tensor, dst=1)
        dist.send(tensor=tensor2, dst=1)
    else:
        # Receive tensor from process 0
        dist.recv(tensor=tensor, src=0)
        dist.recv(tensor=tensor2, src=0)
    print('Rank ', rank, ' has data ', tensor[0])
    print('Rank ', rank, ' has data2 ', tensor2[0])

观察有问题的代码与测试样例的不同

出现报错的代码如下：

if local_rank == 0:
    datas, targets = train_set[0]
    print(targets)
    targets = torch.tensor(5)
    print(targets)
    print(datas.size())
    print(targets.size())
    dist.send(datas, dst=1)
    dist.send(targets, dst=1)
    print(torch.sum(datas))
    print(targets)
    print("Sent all datasets.")
else:
    datas = torch.zeros((1, 28, 28))
    targets = torch.zeros(1)
    dist.recv(datas, src=0)
    dist.recv(targets, src=0)
    print(torch.sum(datas))
    print(targets)
    print("Received dataset from rank 0.")

经测试，在单独发送datas时可以正常接受，但是datas和targets同时发送时会报错：

问题解决方案

原来是数据发送和数据输入的类型不同导致，发送时targets为long类型，接收时targets使用torch.zeros新建时默认时float类型，因此导致发送和接受对应的字节大小不同，导致问题。

修改为如下代码可以正常运行：

targets = torch.zeros(1, dtype=torch.long)
